Serene is a **free, non-profit** mobile app that helps reduce the symptoms of motion sickness using two science-backed techniques. It is fully bilingual (English + Vietnamese).

> **Disclaimer:** Serene is a wellness aid, not a medical device. It does not cure or treat motion sickness — it *may help reduce symptoms*. Consult a healthcare professional for persistent issues.

---

## Why motion sickness happens

Motion sickness affects roughly **46% of people**. The root cause is **sensory conflict**: your eyes see a still screen while your inner ear senses the vehicle moving. Serene attacks this conflict from two angles.

## Features

### 🎧 Audio Therapy — 100 Hz
Inspired by Nagoya University (Japan) research — the same science behind Samsung's "Hearapy". A pure **100 Hz sine wave** stimulates the otolith organs of the inner ear to help re-synchronize the vestibular system.

- Plays a pre-rendered 100 Hz tone for **exactly 60 seconds** (with fade in/out to avoid clicks)
- Built on `expo-audio`; plays even in silent mode, no background audio needed
- A single session's effect lasts ~2 hours — replay anytime

### 🔵 Motion Cues Overlay
Inspired by Apple's **Vehicle Motion Cues** (iOS 18). Eight animated dots sit at the screen edges and move *opposite* to the vehicle's acceleration, giving your eyes the motion signal they're missing.

| Platform | How it works |
|----------|--------------|
| **Android** | A native Kotlin **foreground service** draws a real system overlay (`TYPE_APPLICATION_OVERLAY`) that works over *any* app — TikTok, YouTube, etc. Reads `LINEAR_ACCELERATION` at ~60 fps. Requires the `SYSTEM_ALERT_WINDOW` permission. |
| **iOS** | Apple blocks third-party system overlays, so Serene provides an **in-app** overlay (`expo-sensors` + `react-native-reanimated`) and guides you to enable Apple's built-in Vehicle Motion Cues. |

## Tech Stack

- **Framework:** Expo SDK 54 (bare workflow) + TypeScript (strict)
- **Navigation:** Expo Router v6 (file-based routing in `app/`)
- **Package manager:** pnpm
- **i18n:** i18next + react-i18next + expo-localization (VI + EN)
- **Fonts:** Manrope (`@expo-google-fonts/manrope`)
- **Animation:** react-native-reanimated v4
- **Audio:** expo-audio (pre-rendered 100 Hz WAV)
- **Android native:** Kotlin (`MotionCuesService`, `MotionCuesModule`) via an Expo config plugin

## Project Structure

```

assets/           Icons, splash, 100 Hz audio

```

## Getting Started

### Prerequisites

- **Node.js** 20+
- **pnpm** 10+ (`npm install -g pnpm`)
- **Xcode** 16.3+ (iOS) and/or **Android Studio** (Android)
- A physical device is recommended — motion sensors don't work in simulators

### Install

```bash
pnpm install
```

### Run

```bash
pnpm start          # Start the Expo dev server
pnpm run ios        # Build & run on iOS
pnpm run android    # Build & run on Android
pnpm run prebuild   # Regenerate native android/ and ios/ projects
pnpm run lint       # Lint
```

> The native `android/` and `ios/` folders are generated by `expo prebuild` and are intentionally **not** committed. Run `pnpm run prebuild` after cloning if you need them.

> **Don't install `expo-cli` globally** (`npm i -g expo-cli`) — it's the deprecated legacy CLI and drifts out of sync with the SDK. Use the project-local CLI via `npx expo <cmd>` (or the `pnpm run` scripts above). Add Expo packages with `npx expo install <pkg>`, not `pnpm add`, so versions match the installed SDK.

### Debugging

Use **React Native DevTools** (the only debugger supported by RN 0.81 + new architecture):

```bash
pnpm start          # start Metro, then:
# press  j          # opens React Native DevTools
```

- **Sources** tab → `Cmd+P` → open a file (e.g. `app/(tabs)/index.tsx`) → click the line gutter to set a breakpoint, or drop a `debugger;` statement in code.
- **Console** tab → interactive REPL, inspect objects, view `console.log` output.
- On a physical device: shake → Dev Menu → **Open DevTools** (same thing). Reload with `r` in the Metro terminal (or shake → Reload).

> The VS Code / Cursor **React Native Tools** extension (`reactnativedirect` attach) does **not** work with this RN version — Metro reports `"unsupported debugging client"` and breakpoints stay unbound. Use React Native DevTools (`j`) instead.

### The 100 Hz audio asset

A placeholder `assets/audio/100hz.wav` is included. To regenerate a clean tone:

```bash
# with sox
sox -n -r 44100 -c 2 assets/audio/100hz.wav synth 60 sine 100 fade 0.5 60 0.5 gain -3

# with ffmpeg
ffmpeg -f lavfi -i "sine=frequency=100:duration=60:sample_rate=44100" -ac 2 assets/audio/100hz.wav
```

## Internationalization

All user-facing strings go through `t()` and live in [`i18n/locales/en.json`](i18n/locales/en.json) and [`i18n/locales/vi.json`](i18n/locales/vi.json). Always update both files together. Type-safe keys are declared in `i18next.d.ts`.

## Contributing

This is a non-profit project — contributions are welcome. Please:

1. Use TypeScript strict mode (no `any`, no `@ts-ignore`).
2. Import design tokens from `constants/theme.ts`; never hardcode visual values.
3. Route all copy through i18n and keep `en.json` / `vi.json` in sync.
4. Avoid medical claims — use "may help reduce symptoms", never "cure" or "treat".
